请求拦截器

在请求拦截器中我们可以做这些事情：

  1. 在请求之前做一些操作，比如添加请求头

  2. 不走网络请求直接返回一些数据（可用来模拟请求数据）

  3. 终止请求并触发一个错误

  4. 可以给一个请求加锁，先让请求等待，等某个任务或者请求执行完成之后，在解锁让之前请求继续

    请求加锁

    场景：

    后台要求我们在每个请求时都需要携带一个token值（获取token值也是一个接口，当然这个接口不需要待token），且token值会过期，过期之后则需要我们继续先获取token，然后才能继续请求。

思路：

在响应拦截器中判断状态码是否是token失效标识，
如果是:

  1. 先将请求加锁，不然其他网络请求在进来，
  2. 发新的网络请求去请求新token，（注意要防止该请求重新进入拦截器，可以重新创建一个dio对象）
  3. 获取到token后解锁，让暂停的网络请求继续执行
  4. 我们还需要重新发送当前这个网络请求让其带有token标识

缓存拦截器

缓存拦截器一般用在对信息更新的及时性要求不那么严格时使用。
用在get请求中，在一段时间内多次发送请求时，我们将第一次获得的结果保存起来，在接下来的请求中不走网络请求直接从本地返回保存的结果即可。

主要思路如下：
  1. 在响应拦截器中，判断是否需要缓存数据，若需要，则缓存结果数据
  1. 在请求拦截器中判断是否需要使用缓存数据，如果需要，则直接返回缓存数据
